# CLI

The `mongreldb-kit` binary (crate `mongreldb-kit-cli`) wraps the Rust kit for
scripting and code generation.

```sh
mongreldb-kit <command> [args]
```

## Commands

| Command | Purpose |
|---|---|
| `init <path>` | Create a new database directory |
| `check <path>` | Open a database and verify its internal tables exist |
| `doctor <path>` | Open a database and run an integrity check |
| `schema validate <schema.json>` | Validate a schema JSON file |
| `schema print <path>` | Print the stored schema catalog JSON |
| `migrate apply <path> <migrations.json>` | Apply pending migrations |
| `migrate status <path>` | Print applied migration versions |
| `migrate plan <path> <migrations.json>` | Print which migrations would be applied (alias: `dry-run`) |
| `diff <schema.json> <path>` | Compare a code schema to the stored catalog |
| `generate migration <schema.json> --from <path>` | Generate a migration skeleton for drift |
| `generate types <schema.json> --lang <ts\|rust\|python>` | Generate typed row/insert/update definitions |
| `fixture create <path> <tables...>` | Dump selected table rows to JSON |
| `fixture load <path> <fixture.json>` | Load rows from a JSON fixture |

## generate types

Generate language-native type definitions from a schema JSON file (the same
schema shape used by the kit catalog and the conformance fixtures). The output
is written to stdout, so redirect it to a file:

```sh
mongreldb-kit generate types ./schema.json --lang ts     > src/db-types.ts
mongreldb-kit generate types ./schema.json --lang rust   > src/db_types.rs
mongreldb-kit generate types ./schema.json --lang python > app/db_types.py
```

`--lang` accepts `ts` (or `typescript`), `rust` (or `rs`), and `python` (or
`py`); any other value is an error.

For every table the generator emits three shapes:

- **`<Table>Row`** — every column, with nullable columns widened (`| null`,
  `Option<…>`, `Optional[…]`).
- **`<Table>Insert`** — columns you must supply on insert. Columns that have a
  default or are generated are omitted; nullable columns are optional.
- **`<Table>Update`** — every column made optional, for partial patches.

Table names become `PascalCase` type names (e.g. `user_accounts` →
`UserAccountsRow`). Storage types map to each language's idioms — for example
`int64`/`timestamp` become `bigint` in TypeScript, `i64` in Rust, and `int` in
Python; `json` becomes `unknown` / `serde_json::Value` / `Any`; `bytes` becomes
`Uint8Array` / `Vec<u8>` / `bytes`.

Output is generated code: each file starts with a
`Generated by mongreldb-kit. Do not edit.` header. Re-run the command after a
schema change and commit the regenerated file.
